> 配套方案: 留学公司知识库/11-Eduflow Team 多智能体项目/2026-07-01 EduFlow 群聊外显与温备驻留方案 v0.1 §设计三

> 验收覆盖: `tests/unit/test_residency_wake.py` (12 tests)

Phase 4 是 Phase 3 的依赖闭环。本 Phase 把 `last_active_at` /
`last_handoff_at` / `last_wake_at` 三类 stamp 接进真实路径,
否则 sleep 决策永远 idle_age_s = inf,worker 不会被实际 sleep。

## 未覆盖风险

1. **多 worker 并发 wake**: 当前 send 路径串行 wake,不会互相干扰。
   但如果 manager 同时 dispatch 5 个 worker,各自 wake 都需要
   `wake.lazy_wake_timeout_s` + 实际 spawn 时间。超过 `wake_timeout_s`
   即发 ALERT。
2. **ALERT storm 已缓解(P4-C 调后)**: `fire_wake_failure_alert` 内置
   30 分钟 dedup,同 `(agent, kind)` 窗口内只发一次卡片,后续返回
   `deduped=True` 但 audit log 仍写(可 grep 历史)。
3. **`local_facts.touch_heartbeat` 已存在,与 `agent_residency.touch_active`
   是两个独立时钟。前者用于主群可见心跳,后者用于 idle sweep。
   测试已分别覆盖。

## P4 调整点(2026-07-01 rev 2)

老板复核 Phase 4 后要求的调整,已落地:

### ALERT 模板(P4-A)
- 标题:`[ALERT][wake] manager · wake 失败: {agent}`(加 `[wake]` 子类,sender 改 manager)
- 字段顺序:异常类型 → 当前状态 → 影响范围 → 已自动处理 → 下一步 → 需要谁处理 → 需要老板介入(主要信息前置)
- body:异常类型行嵌入北京时间戳;加"下一步"说明 30 分钟 dedup 行为
- template:`no_pane` → yellow(中度),`ready_marker_timeout` / `spawn_failed` → red(严重)

### stamp 阈值(P4-B)
- `record_message_ack`:所有 ACK kind 都 stamp `last_active_at`(含 completed / reconciled / failed_due_to_runtime)
- `task publish-run`:仅 handoff 类 reason(`worker_completed_handed_to_manager` / `delivered_to_user`)stamp `touch_handoff`,不再无脑给每个 publish reason stamp
- `send.py`:仅 warm target stamp `touch_active`(resident / cold 跳过,减少 agent_residency.json 噪声)
- 阈值调参:`warm_idle_timeout_s` 600 → 900,`handoff_buffer_s` 300 → 600,`wake_timeout_s` 60 → 90(给真任务和慢链路充裕 buffer)

### wake_alert 默认行为(P4-C)
- 默认 channel:`config.supervisor_chat_id()` 优先,回退 `config.chat_id()`(ALERT 走维修通道,不刷主群老板通道)
- `no_chat_id` 不再 silent:返回 `{"errno": "ERR_NO_CHAT_ID"}` 便于上游检测
- sender:卡片 header 用 `manager`(正式 ops owner),audit log 作者仍是 `auto_ops`(向后兼容历史 grep)
- 30 分钟 dedup:同 `(agent, kind)` 窗口内不重发

### 配套测试(P4-D)
`tests/unit/test_residency_wake.py` 27 tests,新增覆盖:
- template per-kind(red / yellow)
- 字段顺序回归
- supervisor channel 优先 + main fallback
- no_chat_id errno / send 失败 errno
- 30 分钟 dedup skip / 窗口过期 / per-kind 隔离
- send 仅 warm stamp / resident 不 stamp
- read --ack completed / failed_due_to_runtime stamp
- audit log 作者仍 auto_ops、卡片 sender 是 manager
