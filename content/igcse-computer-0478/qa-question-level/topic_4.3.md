# Topic 4.3 — Linear Search, Binary Search, and Sorting Algorithms
## QA Question Level File

**1. [F]** Trace through a linear search on the list [3, 8, 2, 7, 5] to find the value 7. How many comparisons are made?

**2. [F]** State two conditions required for binary search to be used on a data set.

**3. [F]** Why is binary search faster than linear search for large sorted data sets?

**4. [S]** Trace the first three passes of bubble sort on the list [5, 3, 8, 1] and show the list after each pass. Circle any swaps made.

**5. [S]** Explain why bubble sort may complete early if no swaps are made in a pass. What is this called?

**6. [S]** A list has 1000 elements. What is the maximum number of comparisons linear search would make? What is the maximum for binary search?

**7. [C]** Describe a trace of insertion sort on [4, 2, 7, 1, 5] showing the list state after each element is inserted into the sorted portion.

**8. [C]** Compare the time complexity of bubble sort and insertion sort in the average case. Which algorithm is generally more efficient and why?

**9. [C]** A programmer claims binary search is always better than linear search. Evaluate this claim, giving specific scenarios where the claim is valid and where it is not.
