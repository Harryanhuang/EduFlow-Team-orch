# Topic 6.3 — Data Validation and Verification
## QA Question Level File

**1. [F]** What is the purpose of data validation?

**2. [F]** State three examples of validation rules.

**3. [F]** What is the difference between validation and verification?

**4. [S]** A form requires a UK postcode. Which validation rules should be applied and why is format checking alone insufficient?

**5. [S]** Explain how a check digit works in validation, using the ISBN-10 system as an example.

**6. [S]** Describe how SQL constraints (NOT NULL, UNIQUE, CHECK) are used to enforce data validation at database level.

**7. [C]** A banking system requires a 6-digit PIN. Evaluate which combination of validation rules would provide the most secure entry and explain your choices.

**8. [C]** When a user registers for an online account, analyse why both validation and verification are necessary and give examples of each method used in the registration process.

**9. [C]** A hospital system stores patient ages. Compare the effectiveness of type checking, range checking, and presence checking for this field. Recommend which validations should be implemented and justify your answer.
