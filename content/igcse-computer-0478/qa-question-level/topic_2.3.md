# Topic 2.3 — Floating-Point Numbers
## QA Question Level File

**1. [F]** Express the decimal number 3.75 in binary floating-point form with an 8-bit mantissa and 4-bit exponent.

**2. [F]** State the purpose of normalising a floating-point number.

**3. [F]** Explain the difference between the mantissa and the exponent in a floating-point representation.

**4. [S]** A floating-point system uses 5 bits for the mantissa (including sign) and 3 bits for the exponent. What is the largest positive number representable? Show your working.

**5. [S]** Explain what is meant by overflow and underflow in floating-point representation.

**6. [S]** Describe how normalised form ensures consistency in floating-point representation.

**7. [C]** Convert the decimal number 0.1 to binary floating-point. Explain why some decimal fractions cannot be represented exactly in binary.

**8. [C]** In IEEE 754 single precision, a value is stored as 1 10000001 01000000000000000000000. Find the decimal value. Show your working.

**9. [C]** A floating-point system uses 4-bit mantissa and 4-bit exponent (both using two's complement). Calculate 2.5 × 10⁻² + 1.25 × 10⁻¹ and state whether the result can be represented exactly.
