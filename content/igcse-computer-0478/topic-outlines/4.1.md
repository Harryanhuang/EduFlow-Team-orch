# Topic 4.1 — Algorithm Design and Pseudocode

## Basic Info
- **Topic ID:** 4.1
- **Domain:** Algorithms and Programming
- **Core/Supplement:** Core
- **Weight in exam:** High (Paper 2 algorithm questions)

## Knowledge Points
1. Algorithm definition and properties (finiteness, definiteness, inputs, outputs, effectiveness)
2. Flowcharts: standard symbols (process, decision, input/output, start/end, arrows)
3. Pseudocode conventions (assignment, input/output, selection, iteration)
4. Sequence, selection (IF-THEN-ELSE), iteration (WHILE, FOR, REPEAT-UNTIL)
5. Tracing algorithms by stepping through with test data
6. Identifying errors in pseudocode (syntax, logic)
7. Writing algorithms for simple problems

## Key Pseudocode Structures
```
INPUT variable
OUTPUT variable
variable ← expression
IF condition THEN ... ELSE ...
WHILE condition DO ... ENDWHILE
FOR counter FROM start TO end DO ... ENDFOR
REPEAT ... UNTIL condition
```

## Common Errors
- Confusing WHILE and REPEAT-UNTIL (pre-test vs post-test loop)
- Off-by-one errors in FOR loops
- Using ← in wrong direction or using = for assignment
- Not initialising variables before use
- Infinite loops (forgetting to update loop counter)

## Difficulty Band: F:2|S:4|C:3
- F: read simple pseudocode, identify sequence/selection/iteration, trace simple algorithms
- S: write pseudocode for given problem, trace with test data, identify logic errors
- C: design algorithms from problem descriptions, simplify/optimise algorithms, nested structures
