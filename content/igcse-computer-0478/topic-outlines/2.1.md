# Topic 2.1 — Binary and Hexadecimal

## Basic Info
- **Topic ID:** 2.1
- **Domain:** Data Representation
- **Core/Supplement:** Core
- **Weight in exam:** Foundation (appears in both Paper 1 and Paper 2)

## Knowledge Points
1. Binary number system: base 2, digits 0 and 1 (bits)
2. Converting binary to decimal and decimal to binary
3. Hexadecimal number system: base 16, digits 0–9, A–F
4. Converting between binary and hexadecimal (4 bits = 1 hex digit)
5. Converting between decimal and hexadecimal
6. Uses of hexadecimal in computing (memory addresses, colour codes #RRGGBB)
7. Binary addition and overflow
8. Signed binary numbers (unsigned vs signed)

## Key Formulas / Facts
- 1 nibble = 4 bits; 1 byte = 8 bits
- Binary to decimal: sum of bit × 2^position
- Hex to binary: each hex digit → 4 binary bits
- Hex to decimal: each digit × 16^position
- Uses of hex: HTML colour codes, memory addresses, assembly language

## Common Errors
- Confusing binary and decimal place values
- Forgetting to carry/borrow in binary arithmetic
- Mixing up the conversion directions
- Not padding binary numbers to groups of 4 bits when converting to hex

## Difficulty Band: F:2|S:4|C:3
- F: binary↔decimal conversion, identify hex digits
- S: binary↔hex conversion, binary addition, uses of hex
- C: multi-step conversions, binary overflow explanation, signed binary introduction
