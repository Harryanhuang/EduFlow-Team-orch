### Question Q-N1-01
**Difficulty**: Foundation
**Question**: Write the first 5 terms of the sequence defined by the nth term formula 3n + 2.
**Answer**: 5, 8, 11, 14, 17
**Explanation**: Substitute n = 1, 2, 3, 4, 5 into 3n+2. A common error is starting with n = 0 (which gives 2, 5, 8, 11, 14) — sequences at IGCSE level typically start at n = 1 unless stated otherwise.

### Question Q-N1-02
**Difficulty**: Foundation
**Question**: Find the nth term of the sequence: 7, 11, 15, 19, 23, ...
**Answer**: 4n + 3
**Explanation**: Common difference is 4, so the formula starts with 4n. When n=1, 4(1)=4, but the first term is 7, so add 3: 4n+3. A common error is writing 4n+7 (using the first term instead of the adjustment).
**Tags**: sequences, nth-term, arithmetic

### Question Q-N1-03
**Difficulty**: Foundation
**Question**: Is 101 a term in the sequence 5, 9, 13, 17, ...? Show your working.
**Answer**: No. The nth term is 4n+1. Setting 4n+1=101 gives n=25, which is a whole number, so YES 101 is the 25th term. A common error is checking if 101 is divisible by 4 instead of solving 4n+1=101.
**Explanation**: Find nth term (4n+1), set equal to 101, solve for n. If n is a whole number, it's in the sequence. A common error is checking divisibility by the common difference.
**Tags**: sequences, nth-term, membership-test

### Question Q-N1-04
**Difficulty**: Standard
**Question**: Find the lowest common multiple (LCM) of 12 and 18 using prime factorisation.
**Answer**: 36
**Explanation**: 12 = 2² × 3, 18 = 2 × 3². LCM = 2² × 3² = 4 × 9 = 36. Alternatively list multiples: 12, 24, 36... and 18, 36... LCM = 36. A common error is multiplying the numbers (216) instead of finding the LCM.
**Tags**: lcm, prime-factorisation, number-theory

### Question Q-N1-05
**Difficulty**: Standard
**Question**: Classify each number as prime, square, or cube: 27, 49, 13, 64, 17.
**Answer**: 27 = 3³ (cube), 49 = 7² (square), 13 (prime), 64 = 4³ = 8² (both square and cube), 17 (prime). A common error is not recognising 64 is both a square and a cube number.
**Explanation**: Prime: only divisible by 1 and itself. Square: n² for some integer. Cube: n³ for some integer. A common error is not checking for both square and cube (64 = 8² = 4³).
**Tags**: classification, prime, square, cube

### Question Q-N1-06
**Difficulty**: Standard
**Question**: Express 84 as a product of prime factors using index notation.
**Answer**: 2² × 3 × 7
**Explanation**: 84 = 2 × 42 = 2 × 2 × 21 = 2 × 2 × 3 × 7 = 2² × 3 × 7. Use a factor tree or repeated division. A common error is stopping at 4 × 21 without further factorising 4 and 21.

### Question Q-N1-07
**Difficulty**: Standard
**Question**: Find the HCF and LCM of 60 and 84.
**Answer**: HCF = 12, LCM = 420
**Explanation**: 60 = 2² × 3 × 5, 84 = 2² × 3 × 7. HCF = 2² × 3 = 12 (common factors with lowest powers). LCM = 2² × 3 × 5 × 7 = 420 (all factors with highest powers). A common error is confusing HCF with LCM.
**Tags**: hcf, lcm, prime-factorisation

### Question Q-N1-08
**Difficulty**: Standard
**Question**: Write √50 in simplest surd form.
**Answer**: 5√2
**Explanation**: √50 = √(25 × 2) = √25 × √2 = 5√2. Look for the largest square number that divides into 50. A common error is writing √50 = 5√10 or √50 = 25√2.
**Tags**: surds, simplification, square-roots

### Question Q-N1-09
**Difficulty**: Challenge
**Question**: The nth term of a sequence is n² - 3n + 1. Find the first term that is greater than 20.
**Answer**: n=6 gives 36-18+1=19 (not > 20). n=7 gives 49-21+1=29. The 7th term is the first greater than 20.
**Explanation**: Substitute increasing values of n until the result exceeds 20. n=1: -1, n=2: -1, n=3: 1, n=4: 5, n=5: 11, n=6: 19, n=7: 29. A common error is not checking systematically.
