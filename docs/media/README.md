# Media Assets

This directory stores project visuals used by README and supporting documents.

## README Visual Set

| Asset | Purpose |
| --- | --- |
| `readme-command-center.svg` | opening project identity and command-center overview |
| `readme-runtime-map.svg` | layered runtime architecture map |
| `readme-delivery-loop.svg` | intake-to-crystallization operating loop |
| `readme-feature-runtime.svg` | runtime lifecycle capability panel |
| `readme-feature-messaging.svg` | messaging and command-control capability panel |
| `readme-feature-workflow.svg` | workflow registry capability panel |
| `readme-feature-trust.svg` | runtime trust capability panel |
| `readme-feature-memory.svg` | durable memory capability panel |
| `readme-feature-education.svg` | education production capability panel |

The README references SVG files directly because they are small, crisp on GitHub, and reviewable in diffs.

## Removed Legacy Visuals

The previous README screenshots, PNG exports, QR image, and older architecture diagrams were removed. New public-facing documentation should use only the `readme-*` SVG set above unless a future redesign intentionally introduces a new asset family.

## Design Notes

- Keep labels short enough to read at GitHub README width.
- Prefer operational diagrams over decorative marketing art.
- Use neutral, high-contrast colors that work on GitHub's light page background.
